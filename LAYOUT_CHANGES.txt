MASTERFLOW RECEIVER AND QUEUE OPERATIONS - ROUTED APPROVAL WORKFLOW

This patch stays inside the Receiver and Queue Operations workstream.
It does not edit requester intake, routing logic, templates, store.js, layout.js,
admin pages, reporting pages, or freight files.

MODIFIED FILES

1. assigned-work.html
   - Removes direct Approve and Reject buttons from the ticket receiver workspace.
   - Adds a receiver-only Send for approval action.
   - Adds an approval summary showing the required manager/director route and current state.
   - Preserves claim, assign, start work, request information, add update, resolve,
     reopen, Receiver Brief, timeline, and editable comments.

2. assets/js/pages/assigned-work.js
   - Determines manager versus director approval using the configured
     directorApprovalThreshold (prototype default: $1,000).
   - Lets receivers validate and send an approval-required request to the approver.
   - Changes the ticket to Awaiting approval and records approval metadata and history.
   - Prevents the ticket receiver from approving or rejecting the request.
   - Shows Awaiting approver while the decision is pending.
   - After approval, shows Approved - Ready to fulfill and allows the receiver to begin fulfillment.
   - Treats rejection as a final decision.
   - Preserves editable comments and the existing Work Center behavior.

3. ticket-queues.html
   - Adds a Pending approvals section to Queue Manager.
   - Explains that Queue Manager access represents authorized approver access in the prototype.
   - Preserves existing queue health, workload, SLA, ownership, gap, and feedback views.

4. assets/js/pages/receiver-feedback.js
   - Adds manager/director approval routing and approval decision handling.
   - Lets the authorized Queue Manager approve, reject, or request more information.
   - Approved requests return to the fulfillment team as Approved - Ready to fulfill.
   - Rejected requests are final and retain the decision reason.
   - Keeps raw diagnostic objects hidden from receiver-facing information.
   - Preserves the Receiver Brief, feedback contract, and Queue Manager analytics.

5. assets/css/WORK_CENTER_APPEND.css
   - Adds approval routing, pending approval, and decision card styles.
   - Preserves existing Work Center, Receiver Brief, Queue Manager, and editable comment styles.

PROTOTYPE AUTHORIZATION NOTE

Queue Manager page access represents an authorized manager/director approver in the prototype.
A production deployment would enforce approver scope through SSO roles, department ownership,
dollar thresholds, and separation-of-duties rules. A requester or fulfillment receiver should
not be able to approve their own request.
